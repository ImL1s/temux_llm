Personal computing began in the mid 1970s with kit machines like the Altair 8800 and the Apple I. The IBM PC, introduced in 1981, established a de facto open hardware standard that allowed many independent vendors to manufacture compatible machines, and over the next two decades the personal computer became a household appliance. The shift from desktop towers to laptops in the 1990s, and from laptops to smartphones in the 2010s, each reduced the size and power budget of mainstream computing devices by roughly an order of magnitude. By 2020, a flagship smartphone integrated a heterogeneous system on a chip with high-performance CPU cores, energy-efficient cores, an integrated GPU optimized for graphics and general purpose compute, an image signal processor, and a dedicated neural processing unit, all sharing a unified memory architecture.

The transformer architecture, introduced in the paper Attention Is All You Need in 2017, replaced the recurrent layers of earlier sequence models with a stack of self-attention and feed-forward blocks. Because every position attends to every other position, transformers parallelize well on accelerator hardware, and they scale predictably with parameter count and training compute. From 2018 onward, large language models based on the transformer architecture grew rapidly: BERT, GPT-2, T5, GPT-3, PaLM, LLaMA, Mistral, Gemma, Qwen, DeepSeek, and many others. Most modern open-weight models are decoder-only transformers trained with a next-token prediction objective on trillions of tokens of text and code.

Edge inference is the deployment of a trained model on a user device rather than in a remote data center. The motivation is fourfold: latency, because round-trips over the network add hundreds of milliseconds; privacy, because raw user prompts and responses never leave the device; cost, because the device owner already paid for the silicon; and offline availability, because mobile networks are unreliable. Edge inference is challenging because the model has to fit in a few gigabytes of memory, run within a thermal envelope of a few watts, and produce tokens fast enough for an interactive user. Quantization to 4 or 8 bit weights, weight sharing across attention heads, and operator fusion are the standard techniques to bring a multi-billion-parameter model into a phone-class budget.

LiteRT, formerly known as TensorFlow Lite, is Google's runtime for executing machine learning models on edge devices. LiteRT-LM is the language-model-oriented variant. It loads a packaged model file with a custom format, builds a graph of accelerated kernels, and exposes a small CLI driver as well as language bindings for Android, iOS, and desktop. The runtime supports a CPU backend through XNNPack, an OpenCL-based GPU backend on Android, a WebGPU backend for browsers, and a vendor-specific NPU path on supported silicon. The accelerator and the kernel set are loaded as shared libraries at runtime, which means a single binary can target devices with very different hardware as long as the dependent libraries are available.

Snapdragon 8 Elite for Galaxy is the high-end mobile system on a chip used in the Samsung Galaxy S25 series. It pairs Qualcomm's third-generation custom Oryon CPU cores with the Adreno 830 GPU and the Hexagon NPU. The Adreno 830 supports OpenCL 3.0 with subgroup operations and large local memory, which makes it well suited to the dense matrix kernels that dominate transformer inference. The Hexagon NPU is reachable through Qualcomm's QNN runtime and the QAIRT toolchain, but a vendor-neutral runtime such as LiteRT-LM typically exposes only the CPU and GPU paths until a specific NPU plugin is loaded.

In a typical CLI invocation, a user pushes the binary, the accelerator shared libraries, and the model file to a device path such as /data/local/tmp, sets LD_LIBRARY_PATH so that the linker finds the runtime libraries, and then invokes the binary with a flag selecting the backend and the path to the model. The binary parses the model header, builds the inference graph, registers the requested accelerator, runs a prefill phase over the input prompt, and then enters a decode loop that emits one token per step until an end-of-sequence token or a user-supplied limit is reached. The runtime usually prints a small benchmark report with the time to first token, the prefill throughput, and the decode throughput.
